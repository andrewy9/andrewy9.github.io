CSS Positioning

Upload Date: 7th September 2020

CSS (Cascading Style Sheet) is a great tool to use for not just adding styles to your boring old html files, but also for positioning your content in such a way that they are easier to engage with and navigate around. Unfortunately with CSS, you cannot simply drag and drop elements on the HTML as you see fit, but you can apply different sets of rules to help the contents of the page organise themselves in the way you want to see. There are many sets of those rules, and the one we will be visiting this time is the relative “ CSS Positioning”.

Before we jump onto the boat…

Step 0 we should take before going into CSS positioning is to look at how the elements on a webpage are presented. All elements are blocks of rectangular blocks with its own pixels of borders and margins.

There are elements such as div that are display: block by default (meaning don’t jump onto the same row as the other elements), and there are elements such as `span` that are `display: inline` by default and will sit horizontally next to other elements.

Now we understand each element has their own boxes of space that they fill up on a webpage, now let’s have a look at their positioning.

Static Positioning

The static positioning is the default positioning for the elements. It is sometimes used to change the element’s positioning back to default when affected by the outside elements.

Relative Positioning

What if I move this element 10px to the right, relative to where it is now? Relative positioning answers that question for you on the webpage, allowing you to move the element in relation to where it would be if it was position: static .

Absolute Positioning

What if you wanted to position something, wherever you wanted, without affecting positions of other elements? Absolute positioning allows you to place an element anywhere within its parent, as if you are sticking the element anywhere you want like a sticker, given the parent element is either relatively or absolutely positioned. If there are no parent elements with absolute or relative positioning, it positions itself in relation to the html.

Figure 8 shows “Div one” with position: absolute . Notice how it is positioned on top of other elements without affecting their position.

Fixed Positioning

Very similar to absolute positioning, except that it is always positioned in relation to the viewport. Because the viewport doesn’t move or change when the window is scrolled, it also means the element with a fixed position doesn’t move when the window is scrolled. This means it can be a useful positioning tool for nav bars.

Figure 11 shows “Div one” with a fixed position with top: 0px. The “Parent” element is positioned relatively but unlike absolute positioning, it is fixed only to the viewport.

Figure 12 shows how scrolling the window does not change the position of the “Div one” element while it is in fixed position.

Sticky Positioning

Sticky Position is like a fixed position where the element is placed like a static positioned element until it has been scrolled past. Given that the parent element has extra room, the sticky element will behave as a fixed positioned element.
